# 成长日记
> child-growth-diary
> 将来自小米摄像机的监控记录转为图片，并进一步生成延时摄影视频

从孩子出生后，家里借助小米摄像机存下了好多监控。但500GB的体积显然无法长期存储，所以考虑每分钟/每天截取一张，做成视频进行保存。这个项目就是相关脚本

# 操作步骤

1.  克隆该项目，执行`pnpm install`安装依赖
2.  获取一系列文件名格式为`YYYYMMDDHHmmss_YYYYMMDDHHmmss.mp4`的监控视频文件，存放于`input`文件夹中
3.  执行`pnpm monitor-video-2-img`, 从input内的视频文件中，每60s截取一张图，输出到`output`文件夹中。命名格式为`${原视频名}_${第n个生成文件}.jpg`
4.  执行`pnpm organize-img-files`, 将`output`中截取出的图片文件，按文件前缀所在日期，规整到文件夹中
5.  执行`pnpm screenshot-2-video`, 将`output`中的图片，按文件名顺序合并为视频


# 其他说明

预期只要文件名格式符合`YYYYMMDDHHmmss_YYYYMMDDHHmmss.mp4`，即可使用该方案。

## 转换计划

1. 每个月从路由器中下载一次监控数据，保存监控数据，并腾出路由器空间
2. 处理原始监控数据，以每60s截取一张图的频率，将原视频转换为图片
3. 将产出图片，通过[monitor-video-2-img](./src/monitor-video-2-img.js), 合成回视频。可以选择每天一张图，或者每分钟一张图两种模式

## 视频转图片方法

在WSL-Ubuntu下调用ffmpeg实现，shell脚本见[monitor-video-2-img](./src/monitor-video-2-img.js)，每次并发10个任务进行。实测每小时可处理2天的数据
