# 小米摄像机记录 => 延时图片

从孩子出生后，家里借助小米摄像机存下了好多监控。但500GB的体积显然无法长期存储，所以考虑每分钟/每天截取一张，做成视频进行保存。这个项目就是相关脚本

# 操作思路

0.  克隆该项目，执行`pnpm install`安装依赖
1.  获取一系列文件名格式为`YYYYMMDDHHmmss_YYYYMMDDHHmmss.mp4`的监控视频文件，存放于`input`文件夹中
2.  

| 是否关键 | 事项 | 目标 |
| :------- | :--- | :--- |


存放小米摄像机记录

可通过以下脚本，小米摄像机记录到的监控视频，将视频以每60s一张图的方式，抽取成图片，并进一步合并为新视频

# 转换计划

1. 每个月从路由器中下载一次监控数据，保存监控数据，并腾出路由器空间
2. 处理原始监控数据，以每60s截取一张图的频率，将原视频转换为图片
3. 将产出图片，通过[img2video](./img2video.js), 合成回视频。可以选择每天一张图，或者每分钟一张图两种模式

# 视频转图片方法

在WSL-Ubuntu下调用ffmpeg实现，shell脚本见[video2img.sh](./video2img.sh)，每次并发10个任务进行，每小时可处理2天的数据
